# 🔔 모던 자바스크립트 Deep Dive 1장~5장

**1장 프로그래밍이란?**

```
1-1. 프로그래밍이란?
    - 컴퓨터에게 실행을 요구하는 일종의 커뮤니케이션.
    - 정확하고 상세한 요구사항을 설명하는 작업 (코드를 통하여).
    - 문제해결능력이 요구됨.

1-2. 프로그래밍 언어
    - 프로그래밍언어(JS)를 사용하여 프로그램을 작성하고,
      인터프리터가 프로그래밍 언어를 기계어로 변환하여 컴퓨터에게 전달한다.
```

**2장 자바스크립트란?**

```
2-1. JS는 현재 모든 브라우저의 표준 프로그래밍 언어가 되었다.

2-3. 자바스크립트 성장의 역사
    - 렌더링은 HTML,CSS,JS로 작성된 문서를 해석해서
      브라우저에 시각적으로 출력하는것 ( CSR, SSR )

2-5. 자바스크립트의 특징
    - 자바스크립트는 인터프리터 언어이다.
    - 컴파일러언어와 인터프리터 언어의 차이점
        컴파일러언어 : 코드가 실행되기 전 컴파일 타임에 코드전체를 머신코드로 변환.
                      실행 전 한번만 수행되고 컴파일이 끝나면 파일이 하나 생성됨.
                      컴파일단계와 실행단계가 분리되어 있어 실행속도가 빠르다.
        인터프리터언어 : 코드가 실행되는 런타임에 코드 한줄씩 바이트코드로 변환.
                        매 실행마다 수행되고 별도의 파일이 생성되지 않는다.
                        인터프리트단계와 실행단계가 분리되어 있지 않아 실행속도가 느리다.
```

**4장 변수**

```
4-1. 변수란?
    - 변수는 하나의 값을 저장하기 위해 확보한 메모리 셀의 메모리 주소에 붙히는 이름.

4-2. 식별자
    - 식별자는 고유해야 하며, 값이 아닌 메모리 주소를 기억하고 있다.
    - 변수명,함수명,클래스명 모두 식별자이다.

4-3. 변수 선언
    - let과 const를 사용한다.
        변수 선언 : let score;
        상수 선언 : const score;  // 선언만 하고 값이 할당되지 않으면 undefined

4-4. 변수 선언의 실행시점과 호이스팅
    - 변수 선언은 소스코드가 인터프리트 단계에 들어가기 전에,
      자바스크립트 엔진이 소스코드의 평가 과정을 거치면서 모든 선언문들이 셋팅된다.
    - 이것을 변수 호이스팅이라고 칭한다.

4-5. 값의 할당
    - 변수 선언은 인터프리트 단계 전에 실행되지만 값의 할당은 인터프리트 단계에서 실행된다.
    - 값이 할당되는것은 이전 값 undefined가 저장된 메모리주소에 새로운 값이 덮어씌워지는것이 아닌
      새로운 메모리 주소에 값이 들어가는 것이다.

4-6. 값의 재할당
    - 재할당도 새로운 메모리 주소에 값이 할당되는 것이고,
      이전 값이 담긴 메모리 셀은 '가비지 콜렉터'에 의하여 메모리가 해제된다.
      < 매니지드언어 : 개발자가 의도적으로 메모리 주소를 정할수도, 삭제할 수도 없다. >

4-7. 식별자 네이밍 규칙
    - 네이밍 컨벤션을 사용한다
        네이밍 컨벤션 : 식별자는 문자,숫자,언더바,달러를 포함할 수 있다.
                       단, 숫자로 시작할 수는 없다.
                       식별자는 예약어와 중첩될 수 없다.
                       대소문자가 구분된다. ( count랑 Count랑 다름 )
                       변수,함수명은 camelCase로 작성한다.
                       생성자함수,클래스는 PascalCase로 작성한다.
```

**5장 표현식과 문**

```
5-1. 값
    - 값은 표현식이 평가되어 생성된 결과물.

5-3. 표현식
    - 표현식은 값으로 평가 될 수 있는 문이다.
        100, 'hello', true, 1+2, x=3
        ( 선언이 되었다는 가정하에 )
        score;, person.name, person.getName(), add()

5-4. 문
    - 프로그램을 구성하는 기본단위이자, 최소 실행단위 이다. ( 명령문이라고도 함 )
    - 선언문, 할당문, 조건문, 반복문 등으로 구분할 수 있다.

5-5. 세미콜론과 세미콜론 자동 삽입 기능
    - 세미콜론(;)은 문의 종료를 나타낸다.
    - { } 뒤에는 세미콜론을 붙히지 않는다. ( 자체 종결성 때문 )

5-6. 표현식인 문과 표현식이 아닌 문
    - 할당문은 표현식인 문
    - 선언문,조건문,반복문은 표현식이 아닌 문
```
