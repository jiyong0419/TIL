# 🔔 모던 자바스크립트 Deep Dive 19장(4)

**19장 프로토타입**

```
19-10. instanceof 연산자
    - instanceof 연산자는 좌항의 객체가 우항의 생성자함수의 instance인지 확인 후 boolean값을 반환한다.
    - 즉, instanceof 연산자는 좌항의 프로토타입 체인에 우항 생성자 함수가 있다면 true,
    없다면 false를 반환한다.

19-11. 직접상속
    - Object.create()메서드
      > 명시적으로 프로토타입을 지정하여 새로운객체를 생성한다.
      > 첫번째 parameter로는 생성할 객체의 프로토타입이 될 객체를 전달한다.
      > 두번째 parameter로는 생성할 객체의 프로퍼티 key와 프로퍼티 디스크립터 객체로
        이루어진 객체를 전달한다. (두번째 parameter는 옵션이다.)
      > 프로토 타입을 지정하며 객체를 생성할 수 있고,
        객체리터럴에 의해 생성된 객체도 프로토타입을 지정할 수 있다는 장점이 있다.
      > ES6에서는 객체 리터럴 내부에서 __proto__접근자 프로퍼티로 직접 상속을 구현할 수 있다.

19-12. 정적 프로퍼티/메서드
    - 정적 프로퍼티/메서드는 생성자 함수로 인스턴스를 생성하지 않아도 참조/호출 할 수 있다.
    - 정적 프로퍼티/메서드는 생성자 함수가 생성한 인스턴스로 참조/호출 할 수 없다.
      오로지 생성자 함수로 다이렉트 참조/호출 할 수 있다.
    - 인스턴스는 생성자함수의 프로토타입 프로퍼티/메서드에 접근할 수 있찌만
      생성자함수의 정적 프로퍼티/메서드는 접근할 수 없다.
    - 인스턴스/프로토타입 메서드 내부에서 this를 사용하지 않는다면 그 메서드는
      정적 메서드로 변경할 수 있다.

19-13. 프로퍼티 존재 확인
    - in 연산자
      > 프로퍼티key in 객체
      > in 연산자는 객체내에 특정 프로퍼티가 존재하는지 여부를 확인한다
      > in 연산자는 객체내부 뿐만 아니라 객체의 모든 프로토타입 프로퍼티도 확인한다.
        > console.log('toString' in person) // true
      > ES6이후 in 연산자 대신 Reflect.has메서드를 사용할 수 있다.
        > console.log(Reflect.has(peson,'toString')) // true
    - Object.prototype.hasOwnProperty 메서드
      > Object.prototype.hasOwnProperty 메서드를 사용해서 객채내에 특정 프로퍼티가
        존재하는지 여부를 확인한다.
        > console.log(person.hasOwnProperty('toString')) // false
      > Object.prototype.hasOwnProperty 메서드는 객체의 프로토타입까지 접근하지는 않는다.

19-14. 프로퍼티 열거
    - for...in문은 객체의 모든 프로퍼티를 순회하며 열거해준다
    - for (변수선언문 in 객체){...}
    - for...in문은 객체의 프로퍼티 개수만큼 순회하며 for...in문의 변수 선언문에서
      선언한 변수에 프로퍼티key를 할당한다.
    - for...in문은 객체의 프로토타입 체인 상에 존재하는 모든 프로토타입의 프로퍼티중에서
      프로퍼티 어트리뷰트 [[Enumerable]]의 값이 true인 프로퍼티를 순회하며 열거한다.
    - for...in문은 프로퍼티key가 심벌인 프로퍼티는 열거하지 않는다.
    - for...in문은 열거할 때 순서를 보장하지 않는다.
    - 배열에는 for...in문을 사용하지말고 for문이나 for...of문, forEach문을 사용하자
    - for...in문은 객체 자신의 고유 프로퍼티뿐만 아니라 상속받은 프로퍼티도 열거한다.
      따라서 Object.prototype.hasOwnProperty메서드를 사용하여 객체 자신의 프로퍼티인지
      확인하는 작업이 불가피하다.
      객체 자신의 고유 프로퍼티만 열거하기 위해서는 Object.keys/values/entries 메서드를 사용하자


```
